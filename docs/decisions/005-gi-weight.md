# ADR-005: GI 의원 페널티 가중 (W_GI_MULTIPLIER)

**날짜**: 2026-04-27
**상태**: 채택 (W=2.0 기본값)

## 배경

마스터 콘셉트는 **소화기내과 전문 + 내시경**. 같은 1.5km 반경에 일반 내과 5개와 GI(소화기) 의원 5개가 있을 때 체감 경쟁 강도는 다르다 — GI 의원은 마스터의 직접 경쟁자, 일반 내과는 부분 경쟁자. 그러나 기존 `c_raw` 산식은 모든 내과 의사 수를 동등 가중으로 합산.

## 데이터 검증 (선결 작업)

`is_gi` 식별 방법 후보:
1. **HIRA 기본정보 API의 의원명 매칭** ("소화기" 키워드): 한국 의원명에 0건 — 사용 불가
2. **HIRA OpenAPI 진료과목/장비 endpoint** (`MadmDtlInfoService2`): 별도 신청 필요
3. **HIRA 의료장비 상세 현황 데이터셋** (data.go.kr 15051055): 711k행 CSV 다운로드 — **채택**
4. **외부 데이터** (네이버/카카오/굿닥): 작업량 大, 매칭 누락

채택 근거: (3)은 신청 불필요 + 712k행 + 의원당 장비코드 명시 + ykiho join 가능.

검증 결과 (1899개 내과 의원 대상):
- ykiho 매칭: **98.5%** (1871/1899)
- **위내시경 (A304: 식도·위·십이지장경)** 보유: 81.9%
- **대장내시경 (A320: 결장경)** 보유: 56.1%
- **둘 다 보유 = `is_gi`**: 56.1%

키워드 trap: HIRA 데이터에 "위내시경"·"대장내시경"·"소화기" 단어 0건 — 의학 용어("식도·위·십이지장경", "결장경")로 표기. **반드시 코드(A304, A320)로 매칭.**

## 결정

`is_gi`를 A304 ∩ A320 보유로 재정의 + `W_GI_MULTIPLIER = 2.0` 도입.

```
n_doctors_med_weighted = n_doctors_med + (W_GI - 1) × n_doctors_gi
```

- `n_doctors_med` (보존): 일반 내과 + GI 동등 가중 의사 수 (display + 회귀 비교용)
- `n_doctors_med_weighted` (신규): GI 의사가 W배. c_raw의 의사 수 입력으로 사용
- 동일 가중 적용 4곳: density 분자, 1.5km 반경, 역세권 500m, subcluster max
- `c_raw`, `c_norm`, `score`, `rank` 모두 가중 적용된 값 기반

## 대안

| 안 | 결정 | 사유 |
|---|---|---|
| W=1.0 (페널티 없음) | 기각 | 기존 c_raw — 마스터 콘셉트 미반영 |
| W=1.5 | 기각 | Top30 1개만 교체. 변별력 약함 |
| **W=2.0** | **채택** | Top30 1개 교체 (구로 수궁동 ↔ 금천 독산4동). GI 의사수 평균 ~2배 가중으로 마스터 직관과 일치 |
| W=2.5 | 기각 | Top30 3개 교체. 차이 더 크지만 내시경 보유 자체로 직접 경쟁 강도 2.5배 격차는 과대 |
| 위·대장 둘 다 + 추가 GI 장비 (옵션 B) | 기각 | 정의 복잡, 데이터 추가 추출 필요. 추후 sensitivity 검토 |

## 검증 (ablation)

`scripts/ablation_gi_weight.py` — W ∈ {1.0, 1.5, 2.0, 2.5} 전체 동 c_raw·c_norm·score 재계산:

| W | Top30 overlap | newly_in | dropped | top1 |
|---|---|---|---|---|
| 1.0 | 30/30 | 0 | 0 | 노원 월계2동 |
| 1.5 | 29/30 | 1 | 1 | 노원 월계2동 |
| **2.0** | **29/30** | **1** | **1** | **노원 월계2동** |
| 2.5 | 27/30 | 3 | 3 | 노원 월계2동 |

**검증 동 의사 수 변화 (W=2.0)**:
| 동 | n_doctors_med | n_doctors_med_weighted | 배율 |
|---|---|---|---|
| 강남 역삼1동 | 17 | 30 | x1.76 |
| 강남 역삼2동 | 24 | 52 | x2.17 |
| 강남 삼성1동 | 5 | 8 | x1.60 |
| 강남 대치1동 | 14 | 23 | x1.64 |
| 노원 월계2동 | 1 | 1 | x1.00 |
| 노원 중계1동 | 9 | 19 | x2.11 |
| 성북 삼선동 | 16 | 38 | x2.38 |

**rank 변동 패턴**:
- 노원 월계2동: rank 1 유지 (모든 W). GI 의원 거의 없음 → 가중 효과 X.
- 강남 GI 多 동(역삼·대치·삼성): W↑에서 c_raw 절대값은 커지지만, rank는 약간 *오르거나* 거의 변동 없음 (174→149, 630→621). **percentile_rank_inverted의 특성** — 모든 GI 多 동이 동시에 페널티 받으면서 상대 순위는 GI가 가장 많은 동만 더 떨어지고 중간은 상대적으로 올라감.
- 즉 **W=2.0의 효과는 점수 ranking보다 c_raw 절대값에 더 큼**. 사용자가 score 외에 c_raw를 직접 비교할 때 의미.

## Caveat

1. **위내시경 보유 81.9%, 대장내시경 56.1%** — 한국 동네 내과의 표준 장비 수준. 단독 보유로는 차별화 시그널 약함. 둘 다 보유(56%)도 변별력은 보통.
2. **A304∩A320 = "GI 후보"가 진짜 시술량 큰 GI 전문 의원과 일치하지는 않음**. 일반 내과가 검진용 1대씩 굴리는 경우와 진짜 GI 전문 (시술량 多, 평판) 의원이 같이 잡힘. 진짜 차별화는 시술량·평판인데 데이터로 측정 불가.
3. **percentile 정규화 효과**: rank 변동은 W=2.0에서 1개뿐. ranking 영향은 작고, 진짜 영향은 **c_raw 절대값** 변화. 사용자가 detail 페이지에서 c_raw·n_doctors_med_weighted 직접 보고 판단할 때 의미가 큼.
4. **ablation 실행 시 `data/scored/scores_<date>.parquet`이 W 마지막 값으로 덮어써짐**. ablation 종료 시 자동으로 `W=W_GI_MULTIPLIER` (기본값)로 다시 실행해 정상 상태 복원. 수동 종료(Ctrl+C) 시 비정상 W 상태로 남을 수 있음 — `python -m scoring.pipeline --date <date>`로 재실행하면 복원.
5. **장비 데이터 갱신 주기 = 연 1회**: HIRA가 매년 12월 31일 기준으로 다음 해 2월에 공개. 신규 의원의 GI 분류는 약 1~14개월 지연.

## 후속 검토

- **W 결정**: 답사·상담 데이터 쌓이면 0.1 단위로 fine-tune (1.8~2.2 범위)
- **GI 정의 강화** (옵션 B): A304∩A320 + (ERCP A316 ∪ 에스상 결장경 A305 ∪ 내시경 ≥ 2대) — 시술량 큰 의원 좁힘. 추후 sensitivity.
- **시술량 데이터**: HIRA 비급여 대장내시경 진료비 정보 또는 보험 청구량 데이터 도입 시 W 계산 정교화 가능.
